# In-Memory Store

## Description
Create a central store to hold the state of all stories and epics in memory, serving as the single source of truth for the UI.

## Acceptance Criteria
- [x] `Store.load()` reads all files in `.devstories` and populates maps
- [x] `Store.getStory(id)` returns correct story object
- [x] `Store.getEpic(id)` returns correct epic object
- [x] `Store.getStoriesByEpic(epicId)` returns array of linked stories
- [x] Store emits `didUpdate` event when data changes
- [x] Store handles "orphan" stories (missing epic) gracefully
- [x] Store validates unique IDs (warns on duplicates)
