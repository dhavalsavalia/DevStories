# Implement accessibility labels in board webview

## Description
Add ARIA labels to all interactive elements in the board webview for screen reader support. Focus on making the kanban board navigable via keyboard and screen reader.

## Acceptance Criteria
- [ ] Add `aria-label` to story cards: "Story {id}: {title}, status {status}, size {size}"
- [ ] Add `role="region"` and `aria-label` to columns: "{status} column, {count} stories"
- [ ] Add `aria-label` to search input: "Search stories"
- [ ] Add `aria-label` to filter dropdowns: "Filter by {type}"
- [ ] Add visible keyboard focus indicators (outline/ring on :focus-visible)
- [ ] Manual test: navigate board with Tab key, verify logical focus order
- [ ] Manual test: enable VoiceOver (Cmd+F5), verify card content is announced
