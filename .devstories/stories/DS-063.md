# Validate user input in command handlers

## Description
Add input validation to all command handlers that accept user input. Commands: createStory, createEpic, quickCapture, changeStatus.

## Acceptance Criteria
- [ ] Story title: max 200 chars, no control characters (allow UTF-8)
- [ ] Epic name: max 100 chars, no control characters (allow UTF-8)
- [ ] Sprint name: alphanumeric + hyphens only
- [ ] Validate required frontmatter fields on parse (id, title, status, type)
- [ ] Show helpful validation error messages (tell user what's wrong)
- [ ] Invalid input doesn't create corrupted files (validate before write)
