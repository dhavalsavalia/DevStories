# Real-time validation diagnostics

## Description

DiagnosticCollection that shows validation errors as red squiggles in frontmatter - missing required fields, invalid enum values, malformed YAML.

## Acceptance Criteria

- [ ] Missing required field shows error at frontmatter end
- [ ] Invalid status/type/size shows error on that line
- [ ] Invalid date format (not YYYY-MM-DD) shows error
- [ ] Non-existent epic reference shows warning
- [ ] Non-existent dependency ID shows warning
- [ ] Diagnostics update on file change (debounced 200ms)
- [ ] Diagnostics clear when file is valid

## Technical Notes

- Create `DiagnosticCollection` named 'devstories-frontmatter'
- Subscribe to `workspace.onDidChangeTextDocument` (debounced)
- Also validate on `workspace.onDidOpenTextDocument`
- Use existing `validator.ts` logic, adapt for line-level errors
- Required story fields: id, title, type, epic, status, size, created
- Required epic fields: id, title, status, created
