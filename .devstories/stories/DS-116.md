# Configurable default sprint for quick capture

## Description
Add config option to control whether quick-captured stories go to backlog or current sprint. Default behavior changes to backlog (safer for inbox workflow). Users who prefer stories in current sprint can enable the option.

## Acceptance Criteria
- [ ] Add `quickCapture.defaultToCurrent` boolean option to config.yaml schema
- [ ] Default value is `false` (stories go to backlog)
- [ ] When `true`, stories use `config.sprints.current`
- [ ] Config option is documented in config.yaml comments
- [ ] Existing behavior preserved when option not specified (backlog)

## Technical Notes
- Add to DevStoriesConfig type in `src/types/config.ts`
- Modify `quickCapture.ts` line 177-178 to check config option
- Update config.yaml with new option under `quickCapture:` section
- Add unit test for both config values
