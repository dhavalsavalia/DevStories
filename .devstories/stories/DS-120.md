# JSON Schema definitions for frontmatter

## Description

Create JSON Schema files defining the structure of story, epic, and config frontmatter. This is the foundation for all validation and autocomplete features, and enables future CLI/MCP reuse.

## Acceptance Criteria

- [ ] `schemas/story.schema.json` with all story frontmatter fields
- [ ] `schemas/epic.schema.json` with all epic frontmatter fields
- [ ] `schemas/config.schema.json` with config.yaml structure
- [ ] All field types, enums, required/optional correctly defined
- [ ] Descriptions for each field (used by hover docs)
- [ ] Schema validates against sample stories/epics

## Technical Notes

- Follow JSON Schema draft-07 or later
- Use `$ref` for shared types (date format, ID pattern)
- Enums for type, size can be hardcoded; status comes from config
