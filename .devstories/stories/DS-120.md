# JSON Schema definitions for frontmatter

## Description

Create JSON Schema files defining the structure of story, epic, and config frontmatter. This is the foundation for all validation and autocomplete features, and enables future CLI/MCP reuse.

## Acceptance Criteria

- [ ] `schemas/defs/common.schema.json` with shared definitions (storyId, epicId, dateString, enums)
- [ ] `schemas/story.schema.json` with all story frontmatter fields
- [ ] `schemas/epic.schema.json` with all epic frontmatter fields
- [ ] All field types, enums, required/optional correctly defined
- [ ] Descriptions for each field (used by hover docs)
- [ ] Unit tests validate sample frontmatter against schemas

## Technical Notes

- Config schema already exists at `schemas/devstories.schema.json`
- Follow JSON Schema draft-07
- Use `$ref` for shared types in `defs/common.schema.json`
- Static enums: type (feature|bug|task|chore), size (XS|S|M|L|XL)
- Dynamic enums (status, sprint): use `type: string` - validated at runtime
