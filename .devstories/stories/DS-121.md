# Single-file frontmatter validation

## Description

Validate frontmatter fields within a single story/epic file. Show diagnostics (red squiggles) for errors.

## Acceptance Criteria

- [ ] DiagnosticCollection registered for devstories
- [ ] Validate required fields present (id, title, type, epic, status, size, created)
- [ ] Validate enum values (type: feature|bug|task|chore)
- [ ] Validate size against config.yaml sizes
- [ ] Validate status against config.yaml statuses
- [ ] Validate date format (YYYY-MM-DD)
- [ ] Validate on file open
- [ ] Validate on file save
- [ ] Validate on file change (debounced 300ms)
- [ ] Error messages are clear and actionable

## Technical Notes

- Use VS Code DiagnosticCollection API
- Parse frontmatter with gray-matter (already in use)
- Read config.yaml for dynamic enum values
