# Single-file frontmatter validation

## Description

Validate frontmatter fields within a single story/epic file. Show diagnostics (red squiggles) for errors.

## Acceptance Criteria

- [ ] DiagnosticCollection registered for devstories
- [ ] Validate required fields present (id, title, type, epic, status, size, created)
- [ ] Validate enum values (type: feature|bug|task|chore)
- [ ] Validate size against config sizes
- [ ] Validate status against config statuses
- [ ] Validate date format (YYYY-MM-DD)
- [ ] Validate on file open, save, and change (debounced 300ms)
- [ ] Invalid files still load (lenient mode) - diagnostics show warnings
- [ ] Error messages are clear and actionable with line numbers

## Technical Notes

- Add `ajv` and `ajv-formats` dependencies
- Create `src/validation/frontmatterValidator.ts` - wraps Ajv with schemas
- Create `src/validation/frontmatterDiagnostics.ts` - DiagnosticCollection provider
- Reuse `isInFrontmatter()` from `storyHoverProviderUtils.ts`
- Re-validate open docs when config changes
