# Cross-file frontmatter validation

## Description

Validate frontmatter fields that reference other files - epic references, dependencies, and [[links]].

## Acceptance Criteria

- [ ] Validate `epic` field references existing epic
- [ ] Validate `dependencies` array entries exist
- [ ] Validate `[[ID]]` references in markdown body resolve
- [ ] Validate IDs are unique across all stories/epics
- [ ] Warnings for orphan stories (epic doesn't list them)
- [ ] Diagnostics update when referenced file changes

## Technical Notes

- Extend `FrontmatterValidator` with `validateCrossFile(frontmatter, type, store)`
- Subscribe to `Store.onDidUpdate` to revalidate when refs change
- Validate `[[ID]]` links in markdown body (below frontmatter)
- Debounce cross-file validation for performance
