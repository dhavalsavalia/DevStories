# Autocomplete for enum fields

## Description

Provide autocomplete suggestions for frontmatter fields with known values.

## Acceptance Criteria

- [x] `status:` suggests values from config statuses
- [x] `type:` suggests feature, bug, task, chore
- [x] `size:` suggests values from config sizes
- [x] `sprint:` suggests values from config sprints.sequence
- [x] Suggestions appear when typing in frontmatter YAML
- [x] Suggestions show description/label where available

## Technical Notes

- Create `src/providers/frontmatterCompletionProvider.ts`
- Register with trigger chars `:` and ` ` (space)
- Reuse `isInFrontmatter()` from `storyHoverProviderUtils.ts`
- Read ConfigService for dynamic values

## Decisions

Sprint autocomplete uses `config.sprintSequence` only (not existing story sprints) - validation catches typos, autocomplete provides valid options.
