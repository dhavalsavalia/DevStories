# Sprint picker in tree view with auto-filter

## Description

Move sprint picker from status bar to tree view title bar. Add config option to auto-filter current sprint on load.

## Acceptance Criteria
- [x] Add filter icon button to tree view title bar (next to +)
- [x] Clicking filter icon opens existing pickSprint QuickPick
- [x] Remove click handler from status bar (display-only)
- [x] Add `autoFilterCurrentSprint` config option (boolean, default: true)
- [x] On extension load, if config is true, auto-set filter to currentSprint from config.json
- [x] Status bar tooltip no longer mentions "click to filter"

## Technical Notes

Config in `.devstories/config.json`:
```json
{
  "autoFilterCurrentSprint": true,
  "currentSprint": "sprint-1"
}
```

Files to modify:
- package.json (view/title menu, pickSprint icon)
- src/view/statusBar.ts (remove command)
- src/core/sprintFilterService.ts (add auto-filter on init)
- src/extension.ts (pass config to sprintFilterService)
- schemas/devstories.schema.json (add autoFilterCurrentSprint)
