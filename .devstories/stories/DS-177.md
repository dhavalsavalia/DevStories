# Fix hardcoded size suggestions

## Description

`getSuggestedSize()` returns literal 'S'/'M' strings. Quick capture hardcodes 'M' as default. Breaks if user configures sizes as `['Tiny', 'Small', 'Large']`.

## Affected Files

- `src/commands/createStoryUtils.ts:147-159` - Returns 'S', 'M' literally
- `src/commands/quickCapture.ts:199` - Hardcoded `size: 'M'`

## Acceptance Criteria

- [x] `getSuggestedSize()` uses config.sizes array indices
- [x] Bug → first size, Feature → middle size, Task → second size, etc.
- [x] Quick capture uses middle element of config.sizes
- [x] Works with any custom size array

## Decisions

Changed `StorySize` type from union literal (`'XS' | 'S' | 'M' | 'L' | 'XL'`) to `string` to support arbitrary custom sizes configured in config.json.
